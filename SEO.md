# SEO.md — SEOルールと運用（清掃事業 / WindWoods）

> codex は新規ページ作成・編集時にこのルールを適用する。

## 1. 基本方針
- 民泊清掃は**出張型ビジネス（Service Area Business）**。検索の本命は**ローカル検索（ローカルパック）＋エリア名キーワード**。
- 1ブランド・1ドメイン（windwoods-stayble.com）に権威を集約。各エリアは**サブディレクトリ**（権威分散を避ける）。
- doorwayを避け、各ページは固有の地域コンテンツを持つ（AGENTS.md §3.2）。

## 2. ページごとのメタ設定
- **title**：`{エリア名}の民泊清掃代行｜リネンサプライ付き｜WindWoods` のように、エリア名＋主要キーワード＋ブランド。各エリアで必ず変える。
- **meta description**：そのエリアの強み（BUSINESS.md §12のストーリー）を120字前後で。エリアごとに固有文。
- **OGP**：title/description/画像（logo or 客室写真）を設定。
- 1ページ1テーマ。エリアページのtitle/h1にエリア名を必ず入れる。

## 3. 構造化データ（JSON-LD）
- 各エリアページに **LocalBusiness**（または CleaningService）スキーマを設置。
  - `name`：WindWoods（株式会社Stayble）
  - `areaServed`：そのページの対象エリア（市区町村名）。恵庭は単独ページで扱うため、千歳・北広島の隣接扱いにはしない。
  - `priceRange` / `telephone` / `email`（backoffice@windwoods-stayble.com）
- 料金表は構造化（可能なら Offer / PriceSpecification）。
- パンくず（BreadcrumbList）を設置（トップ→エリア）。

## 4. 地域キーワード・地区名の散布
- BUSINESS.md §12 の**地区名**を、各エリアページの本文・見出し・alt・metaに自然に散りばめる（不自然な羅列は避ける）。
  - 千歳：千代田町・幸町・本町・栄町・清水町・富丘・青葉・信濃・北栄・美々・勇舞・柏台
  - 北広島：共栄・中央・栄町・大曲・輪厚・西の里・東部・美沢・虹ヶ丘・希望ヶ丘・稲穂町（※黄金町・島松は使わない）
  - 小樽：色内・堺町・入船・花園・稲穂・銭函・朝里・朝里川温泉・祝津・手宮・塩谷・住吉町・港町
  - 恵庭：西島松・島松本町・島松仲町・島松東町・島松寿町・恵み野西・恵み野北・恵み野南・恵み野東・恵み野里美・黄金中央・黄金北・黄金南・柏木町・北柏木町・中島町・戸磯・和光町・住吉町・相生町・有明町・文京町・美咲野
- エリア固有の固有名詞もキーワードに：千歳＝ラピダス/半導体/新千歳空港、北広島＝エスコンフィールド/Fビレッジ、小樽＝運河/観光、札幌＝すすきの/大通/インバウンド、恵庭＝ファイターズファーム施設/西島松/恵み野/島松/JR千歳線。

## 5. 内部リンク
- トップ ⇔ 各エリアページを相互リンク。各エリアページのフッターに「他エリア」リンク。
- 公開対象の対応エリアは、トップ・札幌・千歳・小樽・北広島・恵庭。江別は対応エリアから外し、内部リンクを張らない。

## 6. sitemap / robots / noindex
- `sitemap.xml` を生成し全公開ページを登録。`robots.txt` から参照。
- sitemapはトップ、札幌、千歳、小樽、北広島、恵庭を登録する。江別は登録しない。
- 公開準備中・テスト中のページも noindex を徹底。

## 7. ページ速度・技術
- 静的サイトの利点を活かし軽量に。画像は適切に圧縮（WebP等）、遅延読み込み。
- モバイルファースト（BUSINESS.mdの読み手はスマホ中心）。Core Web Vitals を意識。
- 概算計算JS・フォーム/予約埋め込みは描画をブロックしないよう読み込む。

## 8. Googleビジネスプロフィール（GBP）※ユーザー作業・重要
- 出張型ビジネスの集客の本命。**ユーザー側でGBPを作成・最適化**することを強く推奨（codexはサイト側のNAP一貫性を担保）。
- サイト上の **NAP（名称・住所・電話）をGBPと完全一致**させる（表記揺れ禁止）。

## 9. 定期運用チェックリスト（codexに依頼する想定）
- **週次**：リンク切れ確認、フォーム/予約埋め込みの動作確認、概算計算の数値がBUSINESS.mdと一致しているか。
- **月次**：各エリアのtitle/descriptionの見直し、地域の新情報（再開発・新施設）を地域性セクションへ追記、sitemap更新。
- **随時**：BUSINESS.mdが更新されたら該当箇所をLPへ反映。地域の公式発表が変わった場合は該当エリアページとsitemapを更新。
