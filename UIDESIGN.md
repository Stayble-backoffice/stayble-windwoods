# UIDESIGN.md — デザイン仕様（清掃事業 / WindWoods）

> **役割**：WindWoods 各エリアLPの**視覚的一貫性**を担保する仕様書。codex は作成・編集時に必ず従う。
> 旧LP（ww-sapporo-minpaku.com）のデザインを正として抽出。**文章・数字は古いが、デザインの方向性は踏襲する。**

---

## 1. 基本思想
- トーン：**高級感・ミニマル・硬派・プロフェッショナル**。読み手は民泊オーナー＝事業者なのでB2Bの信頼感を最優先。
- **数字を主役にする**（実績・コスト・時間を極端に大きく見せる）。
- 余白を広く、詰め込みすぎない。

## 2. カラーパレット
| 用途 | 色 |
|---|---|
| ベース背景 | ほぼ純黒 `#000000`〜`#0a0a0a` |
| 基本テキスト | ホワイト `#ffffff` |
| サブ・注釈 | ライトグレー `#9aa0a6` 程度 |
| カード背景 | ダークグレー `#1a1a1a`〜`#2a2a2a` |
| アクセント ゴールド | `#c9a45c` 系（ロゴから要サンプリング）。ヘッダーロゴ・本文ロゴ・実績数字。**ロゴはゴールドで統一**（グリーンは使わない） |
| 比較表の自社ハイライト | 白枠＋明るい背景（他社はグレーに沈める） |

> ゴールドの正確なコードはロゴ画像から抽出して確定する。

## 3. タイポグラフィ
- 日本語：太めゴシック（Noto Sans JP Bold〜Black 等）。見出しは大きく強く。
- 数字：見出し以上に特大。単位（室・%・時間・枚・円）は小さく添える。
- 英語サブコピー：細め装飾フォントで補助的に。スペルミス注意（旧LPに `Spporo` 誤記あり→正 `Sapporo`）。

## 4. レイアウト・コンポーネント
- **セクション見出し**：左に縦バー＋大見出し。
- **カード/ピクトグラム**：白い線画アイコン＋ダークカード。
- **POINTカード**：白帯に黒文字「POINT 1〜5」＋ダークカードに見出し・アイコン・説明。
- **比較表**：自社列を白枠でハイライト、他社列をグレーに沈める。
- **数字ダッシュボード**：円グラフ・大きな数字カード。
- **CTAボタン**：白枠ボタン「まずはお見積もり ▶」。繰り返し配置。

## 6. LP構成テンプレート（全エリア共通の黄金フロー）
> ヒーローの地域名と「地域性セクション」だけ差し替える。地域性の中身は `BUSINESS.md §12`。

1. **ヘッダー**：`logo-mark` ＋ 会社全体実績（例「ご契約 約240室」）＋ ナビ（ホーム/他エリア/リネン/消耗品/会社概要/お問い合わせ）
2. **ヒーロー**：地域名入りキャッチ（例「千歳の宿泊を支えるリネンサプライ付き 民泊清掃代行サービス」）＋ 見積もりCTA ＋ モノクロ客室写真
3. **お悩み提起**：3つ（自社で清掃員を抱えるのはコスト的に／遠方で清掃が回せない／毎回コインランドリーは大変）→ `logo-full`「WindWoodsが解決いたします」
4. **【地域性セクション】**：そのエリア固有のストーリー（BUSINESS.md §12）。doorway回避の核。
5. **サービス詳細 POINT 1〜5**：リネン提携／チェッカー二重チェック／部屋別マニュアル／お問い合わせ簡素化／専用ファイル
6. **消耗品オペ比較**：従来アメニティBOX（倉庫集約）との比較
7. **ランドリーコスト訴求**：数字で→「WindWoodsでこのコストが0に」
8. **参考料金表**：当社 vs 他社（間取り別。数字は BUSINESS.md §8.1。税込）
9. **【概算見積もり＋予約】**：§7（このページの目玉）
10. **ご契約までの流れ**：4ステップ（お問い合わせ→現地調査・お見積もり→簡易審査→ご契約締結。最短1週間）
11. **他エリア導線**＋**フッター**

## 7. 概算見積もり＋予約セクション（このLPの目玉）
> 機能仕様の詳細は `AGENTS.md`。ここでは見た目とUXを規定。

### 7.1 見積もりフォーム
- ダークカード内に必須/任意を視覚的に区別（必須に `*`）。入力タイプは項目ごとに最適化（選択式・数値・ラジオ・時刻・日付・自由記入）。
- 「リネンサプライを利用する」を選ぶと、リネン枚数入力（S/D/Q/K等）が展開する。
- 送信ボタン下に「概算（目安）です。正式見積もりは現地調査後に確定します」と小さく明示。

### 7.2 概算結果の表示
- 計算後、**大きな数字**で「1回あたりの請求額（概算）」を表示（数字主役の原則）。内訳（清掃費／リネン費／消耗品費／ゴミ回収費）を小さく添える。
- 「清掃費」とは書かず必ず「**1回あたりの請求額**」と表記。
- 最大収容人数が10名以上の場合は金額を出さず「要見積もり」と表示。

### 7.3 予約導線（概算の直後）
- 概算の下に2つの大きなボタン：**①オンラインで相談（MTG予約）** / **②現地調査を予約**。
- どちらもTimeRexの該当予約ページを埋め込み／遷移。

### 7.4 予約完了画面
- 「ご予約を承りました」＋予約内容の詳細＋「**確認メールが届いているかご確認ください**」。
- **「サービス概要書をダウンロード」ボタン**（`assets/service-overview.pdf`）。
- トーンは丁寧・安心感。

## 8. モバイルファースト（最重要・誤解禁止）
モバイルファーストとは「PC版をレスポンシブで縮めて縦に積むこと」では **ない**。スマホ表示は **PCとは別物として0から設計し直す**こと。民泊オーナーはスマホ閲覧が中心なので、スマホ版こそ主戦場と考える。

- PC用のテキスト・カード・カラム・表・コンポーネントを **そのまま再利用しない**。スマホ用に作り直す
- PCと同じ配置を再現しようとせず、スマホで不要な要素は **取捨選択して削る**
- 何でも縦1列に積むのは禁止。**引き算と、場合に応じた横並び** を使う
- **ハンバーガーメニュー等、スマホ専用のUI** を設計する
- 「PCの副産物の流用」で済ませない。PC用とスマホ用は **根本から異なる設計** と心得る
- 各ページは PC版を整えた後、必ず **スマホ版を別途・専用に作り直す**
- （基本）特大数字がはみ出さないよう可変サイズに。見積もりフォーム・概算表示・予約ボタンは親指で操作しやすいサイズに

2. ヒーロー等で使う写真素材（旧LPの客室写真を流用可か）
